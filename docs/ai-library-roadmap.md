# AI 智能资料库、筛选、收藏与推荐需求文档

> 目标：把 NAS 里已经扫描出来的广播剧、有声书、网课，进一步整理成更好找、更好看、更懂自己口味的私人声音资料库。  
> 实现方式：分批上线，先做可用的基础资料和收藏，再做 AI 识别，最后做相似推荐。

## 1. 背景

现在系统已经能从 NAS 扫描音频、分季展示广播剧、播放分集、显示封面和进度。但广播剧多起来以后，只靠文件夹名和集数不够用，会遇到几个问题：

- 不知道这个剧是什么类型，比如言情、悬疑、恐怖、古风、现代、修仙。
- 不知道配音演员是谁、原著作者是谁、是否完结。
- 首页和详情页介绍不够丰富，看不出“这部剧值不值得听”。
- 收藏以后没有分类，剧多了以后不好找。
- 听到喜欢的作者或相似类型时，不能快速发现同类作品。

这个需求的核心不是做公开社区，也不是做外部内容平台，而是给自己的本地 NAS 内容增加一层“智能整理”和“私人收藏”。

## 2. 总目标

增加以下能力：

1. 专辑资料增强：为广播剧增加简介、配音演员、作者、标签、题材、情感向、完结状态等信息。
2. AI 智能识别：一键分析广播剧标题和文件信息，自动生成标签、简介、配音演员、作者、完结状态等元数据。
3. 高级筛选：支持按题材、感情向、完结状态、标签筛选。
4. 收藏夹：支持收藏广播剧到默认收藏夹或自建收藏夹。
5. 我的收藏搜索：个人主页里能查看收藏夹，并在收藏里搜索。
6. 相似推荐：在广播剧详情页推荐同作者、同配音、同题材、同标签的本地作品。

## 3. 不做范围

第一阶段不做这些：

- 不做公开评论区。
- 不做粉丝互动、饭圈、动态、社区。
- 不做商城、会员、直播、活动。
- 不自动上传 NAS 文件到外部平台。
- 不默认联网搜索全部资料，除非用户明确点击 AI 识别或联网补全。
- 不把 AI 识别结果当成绝对准确，必须允许手动修改。

## 4. 专辑资料字段

### 4.1 Album 新增字段

后续建议把 `Album` 扩展为：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `author` | `string` | 原著作者或课程主讲人 |
| `cast` | `string[]` | 配音演员、主播、CV |
| `summary` | `string` | 剧情简介或课程简介 |
| `genres` | `string[]` | 题材标签，例如悬疑、言情、恐怖、刑侦、古风 |
| `relationship` | `string` | 感情向，例如言情、耽美、百合、无 CP、群像 |
| `audience` | `string` | 组合筛选，例如男男、女女、男女、群像 |
| `finishStatus` | `string` | 已完结、连载中、未知 |
| `sourceTitle` | `string` | 清理前原始标题，方便回溯 |
| `aiMetaStatus` | `string` | 未识别、已识别、需要确认、识别失败 |
| `aiMetaUpdatedAt` | `string` | 上次 AI 识别时间 |

### 4.2 标签建议

默认题材标签可以先内置这些：

- 感情：言情、耽美、百合、无 CP、群像
- 背景：古代、现代、民国、未来、校园、职场、娱乐圈
- 类型：悬疑、刑侦、恐怖、灵异、修仙、玄幻、无限流、权谋、甜宠、虐恋、治愈、沙雕、强强
- 状态：已完结、连载中、未听、在听、听完

筛选项可以先固定，后续再允许用户自定义。

## 5. AI 智能识别

### 5.1 功能入口

AI 识别入口放在三个地方：

1. 广播剧详情页：按钮 `AI 整理资料`
2. 文件页/系列墙：长按或更多菜单里 `AI 整理`
3. 我的页：`资料整理`，可批量整理缺简介、缺标签的专辑

### 5.2 单个识别流程

用户点击 `AI 整理资料` 后：

1. 后端读取专辑标题、文件夹名、分集标题、已有标签、已有简介。
2. 调用便宜模型，例如 DeepSeek。
3. AI 返回结构化 JSON。
4. 后端校验字段，不合格就丢弃或标记为需要确认。
5. 前端显示确认弹窗：
   - 剧情简介
   - 作者
   - 配音演员
   - 题材标签
   - 感情向
   - 完结状态
6. 用户点击保存后写入本地数据。

### 5.3 批量识别流程

批量识别分两种模式：

1. 智能补全：只处理缺资料的专辑，避免浪费钱。
2. 全库重整：重新分析所有广播剧，用于第一次整理 NAS 内容，或用户觉得标签体系需要重做时。

智能补全处理这些专辑：

- 缺 `summary`
- 缺 `genres`
- 缺 `cast`
- `aiMetaStatus = 未识别`

全库重整处理所有广播剧，但要保护用户手动修改过的字段：

- 用户手动改过的简介、标签、作者、配音演员默认不覆盖。
- 如果用户选择 `允许覆盖`，AI 可以重新生成全部字段。
- 全库重整完成后，需要展示“已更新 / 需要确认 / 失败”的数量。
- 每个专辑保存 `aiMetaStatus`，方便下次只处理失败或待确认项目。

批量执行时要有上限：

- 每次最多 10 个或 20 个。
- 显示本次预计调用数量。
- 每个失败不影响其他专辑。
- 失败原因记录到本地。

### 5.4 DeepSeek 全库智能分析

最终目标是让 DeepSeek 成为本地广播剧资料库的“整理助手”。它需要分析所有广播剧，并生成能直接用于首页展示、筛选和推荐的数据。

分析输入：

- 专辑标题
- 原始文件夹名
- 分季信息
- 分集标题列表
- 已有标签
- 已有简介
- 已有作者和配音演员

分析输出：

- 清理后的剧名
- 原著作者
- 配音演员或主要主播
- 剧情简介
- 短简介，供首页卡片使用
- 题材标签
- 感情向
- 主角组合
- 完结状态
- 推荐关键词
- 相似推荐依据，例如同作者、同题材、同配音

DeepSeek 结果会进入三个系统：

1. 首页介绍：显示短简介、标签、作者或主要配音。
2. 筛选系统：使用 `genres`、`relationship`、`audience`、`finishStatus`。
3. 推荐系统：使用作者、配音、标签和推荐关键词计算相似作品。

全库分析入口：

- 我的页增加 `AI 整理全库`
- 文件页增加 `整理所有广播剧`
- 设置里增加 `AI 资料整理`

全库分析状态：

- 未开始
- 分析中
- 等待确认
- 已完成
- 部分失败

全库分析不能静默覆盖所有结果。第一次做完后，前端要给一个确认页：

- 可以按专辑逐个确认。
- 可以批量接受高置信度结果。
- 低置信度结果标记为 `需要确认`。
- 用户可以跳过某些专辑。

### 5.5 DeepSeek 配置

后端环境变量建议：

```text
AI_METADATA_PROVIDER=deepseek
DEEPSEEK_API_KEY=你的 key
DEEPSEEK_MODEL=deepseek-chat
AI_METADATA_BATCH_LIMIT=10
AI_METADATA_ALLOW_WEB_SEARCH=false
```

如果以后要切换 OpenAI、通义千问、智谱等，只需要换 provider。

默认建议先不开联网搜索，只根据本地 NAS 的文件夹名、剧名、分集名推断。后续如果要更准，可以增加 `AI_METADATA_ALLOW_WEB_SEARCH=true`，让 DeepSeek 或后端搜索模块联网查找公开资料。

### 5.6 AI 输出格式

AI 必须返回 JSON，不要返回自然语言长文。

```json
{
  "title": "今日离港",
  "author": "鱼宰",
  "cast": ["风镜", "陶典"],
  "summary": "一段围绕离别、重逢和自我选择展开的现代情感广播剧。",
  "genres": ["现代", "言情", "治愈"],
  "relationship": "男女",
  "audience": "男女",
  "finishStatus": "已完结",
  "shortSummary": "现代情感广播剧，围绕离别和重逢展开。",
  "recommendationKeywords": ["现代言情", "治愈", "离别重逢"],
  "confidence": 0.72,
  "needsReview": true
}
```

### 5.7 风险控制

- AI 可能胡编配音演员，所以默认标记为 `需要确认`。
- 资料来源不明确时，简介可以保存，配音和作者要降低可信度。
- 用户手动改过的资料，AI 不应直接覆盖。
- 批量识别时要可取消。
- AI key 只保存在 NAS 后端环境变量，前端不能暴露。
- 全库分析必须分批执行，不能一次性把几百部剧全部塞进一个请求。
- 全库分析要有任务进度，刷新页面后还能看到状态。
- 如果 DeepSeek 返回不合法 JSON，后端要重试一次，仍失败就标记为识别失败。

## 6. 筛选系统

### 6.1 首页筛选

首页广播剧列表增加多层筛选：

1. 基础分类：全部、古代、现代、修仙
2. 感情向：全部、言情、耽美、百合、无 CP、群像
3. 主角组合：全部、男女、男男、女女、群像
4. 完结状态：全部、已完结、连载中、未知
5. 题材类型：悬疑、恐怖、刑侦、甜宠、虐恋、权谋、无限流、校园、娱乐圈

移动端不要一次铺太多筛选项。建议：

- 首页只展示一行常用筛选。
- 点 `筛选` 打开底部弹窗。
- 弹窗里分组展示：感情向、状态、题材、标签。

### 6.2 筛选逻辑

- 同一组内单选，例如完结状态只能选一个。
- 题材标签可以多选，例如悬疑 + 刑侦。
- 多组之间为 AND，例如 `已完结 + 悬疑 + 现代`。
- 搜索关键词和筛选同时生效。

## 7. 收藏夹

### 7.1 收藏入口

广播剧卡片和详情页增加收藏按钮：

- 未收藏：空心爱心或收藏图标
- 已收藏：实心状态

点击收藏时弹出收藏弹窗：

1. 默认选中 `默认收藏夹`
2. 显示已有收藏夹列表
3. 有 `新建收藏夹` 按钮
4. 可以输入收藏夹名称，例如言情、悬疑、睡前听、朋友推荐
5. 保存后把当前广播剧加入对应收藏夹

### 7.2 收藏夹数据结构

建议新增：

```ts
type FavoriteFolder = {
  id: string;
  name: string;
  albumIds: string[];
  createdAt: string;
  updatedAt: string;
};
```

默认系统自动创建：

- 默认收藏夹

### 7.3 我的页收藏

个人主页增加 `我的收藏`：

- 顶部搜索框：只搜索收藏里的广播剧。
- 收藏夹横向列表：全部、默认收藏夹、言情、悬疑、睡前听等。
- 下方展示收藏内容。
- 收藏内容支持按最近收藏、最近播放、标题排序。

### 7.4 小红书式收藏体验

参考小红书收藏，不做社交，只借交互：

- 点击收藏后弹出底部收藏夹选择。
- 默认收藏到最近使用的收藏夹。
- 支持新建收藏夹。
- 支持取消收藏。
- 支持从某个收藏夹移除，但不删除专辑本身。

## 8. 相似推荐

### 8.1 详情页推荐区域

在广播剧详情页底部增加：

标题：`你可能还想听`

推荐来源：

1. 同作者，例如墨宝非宝的其他广播剧。
2. 同配音演员，例如同 CV 参与的作品。
3. 同题材，例如现代言情、悬疑刑侦。
4. 同收藏夹，例如都被放进“睡前听”。

### 8.2 推荐排序

推荐分数可以先用简单规则：

| 命中项 | 分数 |
| --- | --- |
| 同作者 | +5 |
| 同配音演员 | 每个 +3 |
| 同感情向 | +3 |
| 同题材标签 | 每个 +2 |
| 同完结状态 | +1 |
| 已经听完 | -2 |
| 当前专辑 | 排除 |

第一版不需要复杂算法，本地规则就够用。

### 8.3 推荐卡片

推荐卡片显示：

- 封面
- 标题
- 作者
- 共同标签，例如 `同作者 · 现代言情`
- 播放按钮

点击后进入该广播剧详情。

## 10. 后端 API 草案

### 10.1 AI 识别资料

```http
POST /api/albums/:id/metadata/analyze
```

返回 AI 建议，不直接覆盖：

```json
{
  "metadata": {
    "author": "鱼宰",
    "cast": ["风镜", "陶典"],
    "summary": "...",
    "genres": ["现代", "言情"],
    "relationship": "男女",
    "finishStatus": "已完结"
  }
}
```

### 10.2 保存资料

```http
PATCH /api/albums/:id/metadata
```

### 10.3 批量 AI 识别

```http
POST /api/metadata/analyze-batch
```

请求体：

```json
{
  "mode": "missing-only",
  "kind": "drama",
  "limit": 10,
  "overwriteManualFields": false
}
```

其中 `mode` 支持：

- `missing-only`：只整理缺资料的广播剧。
- `all`：整理所有广播剧。
- `failed-only`：只重试识别失败的广播剧。

### 10.4 全库 AI 分析任务

```http
POST /api/metadata/analyze-library
GET /api/metadata/analyze-library/:jobId
POST /api/metadata/analyze-library/:jobId/apply
POST /api/metadata/analyze-library/:jobId/cancel
```

说明：

- `analyze-library` 创建全库分析任务。
- `GET` 查询任务进度。
- `apply` 保存用户确认后的结果。
- `cancel` 取消任务。

### 10.5 收藏夹

```http
GET /api/favorite-folders
POST /api/favorite-folders
PATCH /api/favorite-folders/:id
DELETE /api/favorite-folders/:id
POST /api/favorite-folders/:id/albums/:albumId
DELETE /api/favorite-folders/:id/albums/:albumId
```

### 10.6 推荐

```http
GET /api/albums/:id/recommendations
```

## 11. 分批实现计划

### 第一批：资料字段和手动编辑

目标：不用 AI，先把数据结构和 UI 打通。

- 扩展 Album 字段：作者、配音演员、简介、题材、感情向、完结状态。
- 详情页显示这些字段。
- 首页卡片显示标签和简介。
- 增加手动编辑资料弹窗。
- 保存到本地 `data/state.json`。

验收：

- 能手动给一个广播剧添加简介、作者、配音演员、标签。
- 刷新页面后资料还在。
- 首页能看到标签和简介。

### 第二批：筛选系统

目标：剧多了以后能快速找到。

- 首页增加筛选入口。
- 支持感情向、主角组合、完结状态、题材类型。
- 搜索和筛选可以同时生效。

验收：

- 选择 `已完结 + 悬疑` 后，只显示符合条件的广播剧。
- 清空筛选后恢复全部。

### 第三批：收藏夹

目标：能像小红书一样收藏到不同收藏夹。

- 新增收藏夹数据结构。
- 详情页和卡片增加收藏按钮。
- 收藏弹窗支持选择默认收藏夹和新建收藏夹。
- 我的页增加我的收藏、收藏夹列表、收藏搜索。

验收：

- 一个广播剧可以收藏到 `默认收藏夹` 或自定义收藏夹。
- 我的页能按收藏夹查看。
- 收藏搜索能搜到已收藏广播剧。

### 第四批：AI 智能识别

目标：用 DeepSeek 一键补全资料，并支持全库整理所有广播剧。

- 接入 DeepSeek。
- 新增 AI 识别接口。
- 单个广播剧 AI 整理资料。
- AI 结果确认弹窗。
- 支持批量识别缺资料专辑。
- 支持 `AI 整理全库`，分析所有广播剧。
- AI 结果进入首页介绍、筛选条件和推荐系统。
- 支持任务进度、取消、失败重试。

验收：

- 点击 `AI 整理资料` 后生成简介、标签、配音演员建议。
- 点击 `AI 整理全库` 后能分批分析所有广播剧。
- 全库分析结束后能看到已更新、待确认、失败数量。
- 首页能展示 AI 生成的标签和短简介。
- 筛选能使用 AI 生成的题材、感情向、完结状态。
- 用户确认后才保存。
- DeepSeek API key 不暴露到前端。

### 第五批：相似推荐

目标：听完一部以后能发现同作者、同配音、同题材的本地作品。

- 后端增加推荐计算。
- 详情页底部显示 `你可能还想听`。
- 推荐理由显示为 `同作者`、`同配音`、`同题材`。

验收：

- 打开某个广播剧详情页，底部能看到相关广播剧。
- 当前广播剧不会推荐自己。
- 已经没有资料的剧不会乱推荐。

## 12. 推荐优先级

建议先做：

1. 第一批：资料字段和手动编辑
2. 第三批：收藏夹
3. 第二批：筛选系统
4. 第四批：AI 智能识别
5. 第五批：相似推荐

原因：

- 资料字段是所有后续功能的地基。
- 收藏夹对日常使用最直接，马上能提升体验。
- 筛选依赖标签和状态。
- AI 识别要等字段稳定后再接。
- 推荐依赖作者、配音、标签足够丰富。

## 13. 后续问题

实现前还需要确认：

1. AI 识别是否允许联网搜索，还是只根据文件名和分集名推断？
2. 配音演员信息如果 AI 不确定，是保存为“待确认”，还是不保存？
3. 收藏夹是否需要排序和置顶？
4. 推荐是否只推荐本地已有作品，不跳转互联网？
5. 网课和有声书是否也要支持同样的 AI 资料识别？

默认建议：

- 推荐只基于本地已有内容。
- AI 结果默认需要确认。
- 第一版先只做广播剧，稳定后扩展到有声书和网课。
