# 私人听书房设计规范

> 参考对象：`advplyr/audiobookshelf`  
> 风格参考：猫耳 FM / M 站的广播剧声音剧场体验  
> 设计目标：连接 NAS 里的本地音频内容，做一个手机优先、封面好看、适合自己和朋友使用的私人广播剧/有声书/网课播放器。

## 0. 用户需求确认

这个产品的核心不是公开内容平台，也不是完整复刻 Audiobookshelf，而是：

- 内容来源：NAS 文件，不依赖外部联网。
- 内容类型：广播剧、有声书、网课。
- 使用人群：自己和少数朋友。
- 主要设备：手机。
- 体验参考：猫耳 FM 那种移动端声音剧场视觉，重视剧集封面、顶部栏目、分集追听、粉紫氛围和睡前沉浸播放。
- 重点能力：直接连接 NAS、浏览文件、自动整理成专辑、漂亮封面、断点续播、倍速、定时关闭、朋友账号。
- 不做：公开社区、饭圈、追星、直播、发现流、商城、会员、广告、客服、电商化推荐、外部内容抓取。

设计优先级：

1. 手机端好用。
2. NAS 文件找得到、播得稳。
3. 封面和专辑页好看。
4. 广播剧/有声书/网课三类内容清楚分开。
5. 给朋友用时足够简单。

## 1. 设计判断

Audiobookshelf 的功能已经很完整：书库、播客、系列、收藏、播放列表、作者、朗读者、上传、元数据、RSS、用户权限、统计、备份、播放器、电子书阅读器等都在同一个系统里。但现有页面主要问题是：

- 视觉过旧：深灰背景、木质书架、硬阴影、黄色高亮和 Material Symbols 组合起来像早期工具软件。
- 信息层级不稳：顶部栏、侧栏、书架工具栏、播放器都很抢注意力，用户很难一眼判断当前最重要的操作。
- 播放器像“控制面板”，不是“正在听的上下文”。章节、书签、速度、剩余时间、睡眠定时都重要，但需要重新分层。
- 书库页拟物感太强，封面本身的美感没有被充分释放。
- 管理功能和聆听功能混在一起，个人使用时会显得重。

新产品不应只是“换皮 Audiobookshelf”，而应变成一个以“听剧、听书、听课”为中心的手机应用。它可以借 Audiobookshelf 的自托管能力，但主视觉更接近猫耳：打开就是好看的封面、清楚的栏目、分集、追听、播放和睡前沉浸。

## 2. 产品性格

设计关键词：

- 猫耳式声音剧场
- 手机优先
- 私人听剧房
- 封面优先
- 本地 NAS
- 长篇续播
- 好找好听
- 朋友也会用
- 轻二次元但不幼稚

不采用：

- 木质书架拟物
- 广告位和运营弹窗
- 会员化、商城化、直播化
- 发现页、热榜、饭圈互动、追星广场
- 客服中心、活动页、任务体系
- 复杂服务器管理入口
- 桌面后台式密集表格
- 过暗的技术工具风

核心设计句：

> 借猫耳的漂亮壳子，装自己的 NAS 内容：广播剧、有声书、网课，打开就能找、能听、能续播。

## 3. 信息架构

### 3.1 一级导航

移动端是主战场。首页顶部采用猫耳式栏目切换，但只保留自用内容分类，不保留直播、发现、商城等互联网栏目。

顶部栏目：

| 名称 | 目的 | 说明 |
| --- | --- | --- |
| 广播剧 | 追剧、分集、番外、花絮 | 默认栏目 |
| 有声书 | 章节、朗读者、完成进度 | 适合长篇书 |
| 网课 | 课时、学习进度、复习标记 | 适合课程 |

底部导航：

| 名称 | 目的 | 说明 |
| --- | --- | --- |
| 首页 | 继续听、最近添加、顶部栏目内容 | 打开后的默认页 |
| 文件 | 直接浏览 NAS 目录 | 找未整理文件 |
| 搜索 | 搜标题、文件夹、演员/作者、课程名 | 本地索引搜索 |
| 我的 | 下载/缓存、收藏、历史、账号、NAS 设置 | 个人中心 |

### 3.2 首页结构

首页不再叫 Home，而叫“首页”或“听”。推荐模块顺序：

1. 继续听
2. 最近添加
3. 最近追剧
4. 广播剧
5. 有声书
6. 网课
7. 待整理文件

“待整理文件”只显示 NAS 新扫到但还没归类的内容。默认不要把管理噪音放在首页顶部。

### 3.3 NAS 文件入口

因为你的内容都在 NAS 里，“文件”必须是一级入口，而不是藏在设置里。

文件页需要支持：

- SMB / WebDAV / NFS 中至少优先支持 SMB 和 WebDAV。
- 添加 NAS 地址、账号、密码、共享目录。
- 浏览目录树。
- 搜索当前目录和全库索引。
- 将文件夹标记为广播剧/有声书/网课。
- 扫描文件夹并生成专辑。
- 手动选择封面。
- 文件改动后增量刷新。

### 3.4 管理区收纳

扫描、用户、权限、备份、索引、转码属于“设置”。普通朋友账号默认看不到。

## 4. 视觉系统

### 4.1 主题方向

默认主题：`Maoer Private`

整体参考猫耳 FM 的页面审美：广播剧优先、封面更漂亮，粉紫色系更强，界面可以有一点二次元轻甜感，但不能变成幼稚玩具风。和猫耳不同的是，这里只借视觉，不借互联网业务：去掉公开社区、直播、发现、会员、商城、客服和运营噪音，让它更像私人 NAS 声音剧场。

默认使用浅色主题，因为手机白天使用更多；播放页和睡眠场景提供深色沉浸模式。

可选深色主题：`Night Drama`

用于睡前听剧、夜间播放和全屏播放器。

### 4.3 颜色规则

- `accent` 只用于“当前正在听 / 可播放 / 当前激活 / 主按钮”。
- 危险操作不能只靠红色，必须配合文案和确认。
- 首页可以有粉紫剧场氛围，但不能变成广告页。
- 广播剧、有声书、网课可以用不同分类色，但主播放色永远是粉色。
- 封面背景可以使用封面取色后的模糊光晕，主要用于详情页和全屏播放页。

## 5. 字体与排版

推荐字体：

- 中文界面：`Noto Sans SC` 或 `Source Han Sans SC`
- 英文界面：`Geist` / `Satoshi` / `Aptos`
- 数字与时间：`JetBrains Mono` 或 `Geist Mono`

字号阶梯：

| 名称 | 尺寸 | 用途 |
| --- | --- | --- |
| `display` | 32/40 | 首页主标题、详情页书名 |
| `title` | 24/32 | 区块标题 |
| `subtitle` | 18/26 | 卡片标题、弹窗标题 |
| `body` | 15/24 | 正文、描述 |
| `meta` | 13/18 | 作者、时长、状态 |
| `caption` | 12/16 | 标签、辅助说明 |
| `time` | 13/18 mono | 播放时间、剩余时间 |

排版规则：

- 书名最多两行，超过后截断。
- 作者、朗读者、系列信息优先一行展示。
- 元数据不要全部堆在详情页首屏，分成“聆听信息”和“文件信息”两组。
- 中文界面行高不低于 1.45。

## 6. 间距与布局

基础间距：4px 网格。

| Token | 值 |
| --- | --- |
| `space-1` | 4px |
| `space-2` | 8px |
| `space-3` | 12px |
| `space-4` | 16px |
| `space-6` | 24px |
| `space-8` | 32px |
| `space-10` | 40px |
| `space-12` | 48px |

页面布局：

- 桌面端内容最大宽度：`1440px`
- 顶栏高度：`64px`
- 左侧导航宽度：`72px` 收起，`240px` 展开
- 播放器高度：迷你 `88px`，完整 `280px`
- 书籍网格最小封面宽度：`128px`
- 书籍网格推荐封面宽度：`160px - 184px`

圆角：

| Token | 值 | 用途 |
| --- | --- | --- |
| `radius-xs` | 4px | 封面、进度条 |
| `radius-sm` | 8px | 输入框、按钮、菜单 |
| `radius-md` | 12px | 弹层、详情面板 |
| `radius-pill` | 999px | 标签、播放按钮 |

封面仍用小圆角，不做大圆角卡片，保留“书”的形状。

## 7. 图标系统

推荐图标库：Phosphor Icons。

原因：

- 线条更柔和，适合阅读/音频产品。
- 有书、播放、列表、用户、设置、时钟等完整语义。
- 比 Material Symbols 更不“安卓系统面板”。

规则：

- 全局只使用一个图标库。
- 默认线宽 `1.75`。
- 导航图标尺寸 `22px`。
- 工具按钮图标尺寸 `18px - 20px`。
- 播放主按钮图标尺寸 `32px - 40px`。

## 8. 核心组件规范

### 8.0 猫耳式内容组件

广播剧、有声书、网课三类内容共用“专辑”模型，但展示重点不同。

广播剧专辑：

- 大封面，允许更强视觉氛围。
- 显示追剧状态：未听 / 追听中 / 已完结。
- 显示分集更新：正剧、番外、花絮、预告。
- 可选支持角色/声线备注，但只作为本地元数据，不做追星、粉丝互动或公开资料页。
- 支持私人笔记和书签，不做弹幕、评论区和社区互动。

有声书专辑：

- 强调作者、朗读者、章节、总时长、完成进度。
- 支持按章节、文件、书签浏览。
- 封面更像书封，不强行二次元化。

网课专辑：

- 强调课程进度、课时、已学/未学、倍速、笔记。
- 支持按目录树展示。
- 支持标记重点课、已掌握、待复习。

### 8.1 移动端标题栏

用途：模拟猫耳顶部栏目结构，但只保留本地内容分类。

结构：

- 顶部第一行：本地搜索框 + NAS 连接状态。
- 顶部第二行：`广播剧 / 有声书 / 网课` 三个栏目。
- 右侧只放必要入口：缓存、设置或用户。

规则：

- 不出现直播、发现、商城、会员、客服。
- 当前栏目用粉色下划线或胶囊高亮。
- 栏目切换不刷新远程内容，只切换本地索引分类。
- NAS 断开时用轻提示，不弹大广告式弹窗。

### 8.2 侧边导航

当前 Audiobookshelf 的 80px 侧栏每项高度很大，但信息仍显挤。新设计：

- 默认窄栏只显示图标。
- hover 或用户固定后展开文字。
- 当前页用左侧 3px accent 指示。
- 管理类入口放到底部。

桌面端可选导航项顺序：

1. 首页
2. 广播剧
3. 有声书
4. 网课
5. 文件
6. 搜索
7. 我的
8. 设置

### 8.3 书籍卡片

卡片不使用厚重容器，封面是主体。

组成：

- 封面
- 进度条
- 标题
- 作者/朗读者
- 状态徽标

hover 状态：

- 封面轻微上浮 `translateY(-2px)`
- 出现播放按钮
- 出现更多菜单
- 不出现大面积黑色遮罩，最多使用 40% 半透明层

状态徽标：

| 状态 | 显示 |
| --- | --- |
| 未开始 | 无 |
| 正在听 | 细进度条 + 当前章节 |
| 已完成 | 小勾标 |
| 缺信息 | 小 warning 点 |
| 已共享/RSS | 低权重图标 |

### 8.4 内容列表

用于播客 episode、章节、文件列表、上传队列。

列表行结构：

- 左：标题和次级信息
- 中：时长、发布日期、进度
- 右：播放/更多动作

规则：

- 行高 `56px - 72px`
- hover 显示动作按钮
- 正在播放的行使用 accent 左边线和轻背景

### 8.5 筛选与排序

筛选条不应像表单堆叠。建议做成一行 compact toolbar：

- 视图切换：网格 / 列表 / 紧凑
- 筛选：状态、类型、作者、标签、语言
- 排序：最近听、添加时间、标题、时长、随机
- 批量选择

移动端筛选使用底部抽屉。

### 8.6 弹窗与抽屉

规则：

- 轻量确认用 modal。
- 复杂编辑用右侧 drawer。
- 移动端 drawer 从底部出现。
- 元数据编辑不使用满屏表单，按“基础信息 / 文件 / 章节 / 高级”分 tab。

## 9. 播放器规范

播放器是本产品最重要的组件。

### 9.1 播放器层级

三种状态：

| 状态 | 场景 | 高度 |
| --- | --- | --- |
| Mini Player | 浏览书库时常驻 | 88px |
| Focus Player | 用户点击正在播放区域 | 280px |
| Full Player | 睡前、移动端、专注听 | 全屏 |

### 9.2 Mini Player

内容：

- 封面缩略图
- 书名
- 当前章节
- 已听/总时长
- 进度条
- 后退 15s
- 播放/暂停
- 前进 30s
- 速度
- 睡眠定时

规则：

- `播放/暂停` 是唯一主按钮。
- 速度、睡眠、章节、书签是次级动作。
- 进度条横跨播放器底部，鼠标悬停时增厚。

### 9.3 Focus Player

增加：

- 章节列表预览
- 书签
- 播放队列
- 音量
- 输出设备/投屏
- 当前章节剩余时间
- 睡眠定时状态

Focus Player 中应把“当前章节”放大，因为听有声书时章节比整本书总进度更有心理意义。

### 9.4 Full Player

移动端和睡前场景优先：

- 大封面
- 大播放控制
- 亮度低
- 可单手点击
- 睡眠定时强入口
- 锁屏式极简信息

推荐手势：

- 左右滑动：切章节
- 双击左侧：后退 15s
- 双击右侧：前进 30s
- 上滑：章节/书签
- 下滑：回到 mini player

## 10. 页面规范

### 10.1 首页

目的：减少选择成本。

模块：

- 继续听：横向 1-3 个大条目
- 睡前继续：按最近夜间播放排序
- 最近加入：封面网格
- 广播剧：大封面横滑
- 有声书：章节进度封面网格
- 网课：课程进度列表
- 本周摘要：听书时长、完成章节、连续天数
- 需要整理：只显示异常

### 10.2 分类栏目页

目的：分别浏览广播剧、有声书、网课。

视图：

- 封面网格：默认
- 列表：管理和筛选用
- 紧凑：大量文件用

默认排序：

- 个人库：最近活动
- 新库：添加时间
- 管理模式：标题

### 10.3 详情页

首屏结构：

- 左：封面
- 中：书名、作者、朗读者、系列、主按钮
- 右：聆听状态、进度、剩余时间、最近播放

下方 tab：

- 章节
- 详情
- 文件
- 笔记/书签
- 管理

主按钮规则：

- 未开始：开始播放
- 已开始：继续播放
- 已完成：重新播放
- 有电子书：播放按钮旁显示阅读按钮

### 10.4 系列页

系列页要体现“长线进度”：

- 系列总封面墙
- 已完成/总数
- 总时长
- 下一本
- 系列内排序
- 缺失项提示

### 10.5 作者/朗读者页

作者和朗读者不应分裂成完全不同体验。建议同页切换：

- 作者
- 朗读者
- 出版社

卡片显示：

- 头像或首字母
- 作品数
- 已听时长
- 最近一本

### 10.6 管理页

管理页视觉可以更像设置后台，但仍保持同一设计系统。

分组：

- 书库
- 用户
- 扫描与匹配
- 元数据
- 播客/RSS
- 备份
- 系统日志

## 11. 交互状态

每个交互组件必须包含：

- default
- hover
- active
- focus-visible
- disabled
- loading
- error
- selected

焦点样式：

- 使用 `2px` accent 外描边。
- 不依赖浏览器默认蓝框。
- 键盘导航顺序必须与视觉顺序一致。

加载状态：

- 封面加载使用骨架，不显示空白卡片。
- 播放器 seek loading 使用局部 loading，不冻结整页。
- 扫描任务用后台任务中心展示，不用全屏阻塞。

## 12. 移动端规范

移动端默认信息架构：

- 顶部栏目：广播剧 / 有声书 / 网课
- 底部导航：首页 / 文件 / 搜索 / 我的
- 播放器常驻底部，在底部导航上方
- 点击播放器进入全屏播放
- 搜索为全屏 modal
- 筛选为底部抽屉

触控尺寸：

- 最小点击区域：44px
- 播放主按钮：64px
- 底部导航项：56px 高
- 进度条可拖动热区：至少 24px 高

## 13. 可访问性

- 所有图标按钮必须有 `aria-label`。
- 颜色对比度正文不低于 4.5:1。
- 播放状态通过文本和图标共同表达。
- 进度条必须支持键盘调整。
- 弹窗打开后焦点进入弹窗，关闭后回到触发按钮。
- 支持 `prefers-reduced-motion`。
- 播放器时间变化不应频繁打断屏幕阅读器。

## 14. 动效规范

动效要轻，像翻页和呼吸，不像游戏界面。

| 场景 | 时间 | 曲线 |
| --- | --- | --- |
| hover | 120ms | ease-out |
| modal/drawer | 180ms | cubic-bezier(.2,.8,.2,1) |
| mini player 展开 | 220ms | cubic-bezier(.16,1,.3,1) |
| 页面切换 | 160ms | ease-out |
| 封面加载 | 220ms | ease-out |

避免：

- 循环发光
- 大面积背景动画
- 过度弹跳
- 播放器频繁重排

## 15. 命名建议

如果这是给自己用的新软件，不建议继续叫 Audiobookshelf。可选方向：

- `Listen Shelf`
- `Night Library`
- `Chapter Room`
- `Echo Library`
- `书听`
- `听书房`

中文界面临时推荐名：`听书房`。

理由：比“有声书架”更像一个使用场景，能覆盖有声书、播客、课程、电子书和个人笔记。

## 16. 设计验收清单

做 UI 原型或开发时，用下面清单验收：

- 打开首页 3 秒内能知道下一本该听什么。
- 正在播放信息在所有页面都可见。
- 播放器的主按钮永远最显眼。
- 书库页面第一眼看到的是封面，不是工具栏。
- 管理动作存在，但不打扰普通听书。
- 深色主题夜间不刺眼。
- 移动端可以单手完成播放、暂停、快进、睡眠定时。
- 缺封面、缺章节、扫描错误等异常能被集中处理。
- 列表、网格、详情、播放器使用同一套 token。
- 所有按钮 hover/focus/disabled/loading 状态完整。

## 17. 下一步落地顺序

建议先做 4 个高保真页面：

1. 今天页
2. 书库页
3. 书籍详情页
4. Mini + Full 播放器

这 4 个页面确定后，再扩展管理页、上传页、统计页和移动端细节。
